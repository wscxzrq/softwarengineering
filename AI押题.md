## 五、简答题预测（5题）

### 【预测题1】简述瀑布模型每个阶段的任务 ⭐⭐⭐

**答题要点：**
1. **需求定义**：弄清用户对软件的全部需求
2. **系统和软件设计**：建立软件总体结构，画出软件体系结构图
3. **实现与单元测试**：用编程语言实现软件，并对其进行测试
4. **集成与系统测试**：将软件所有模块集中到一起进行测试
5. **运行与维护**：软件投入使用，根据用户需求进一步优化软件

---

### 【预测题2】简述模块的耦合性并列举三种耦合类型 ⭐⭐⭐

**答题要点：**
- **定义**：衡量模块间依赖程度的标准，接口复杂，调用方式紧密，传递数据多，耦合性就强

- **七种耦合类型**：
  1. **非直接耦合**：模块间无直接关系，通过主模块控制调用
  2. **数据耦合**：通过简单数据参数交换信息
  3. **标记耦合**：传递数据结构的子结构
  4. **控制耦合**：通过控制信息控制另一模块逻辑
  5. **外部耦合**：访问同一全局简单变量
  6. **公共耦合**：访问同一公共数据环境
  7. **内容耦合**：直接访问另一模块内部数据（耦合度最高）

---

### 【预测题3】简述单元测试的主要内容 ⭐⭐⭐

**答题要点：**
对程序中单个模块进行测试，又称模块测试，模块分为驱动模块与桩模块：
- **驱动模块**：代替被测模块的调用模块
- **桩模块**：代替被测模块所调用的模块

需要分别从以下5方面进行检查：
1. 模块接口
2. 局部数据结构
3. 路径
4. 错误处理
5. 边界

---

### 【预测题4】简述类图中组合关系的含义 ⭐⭐

**答题要点：**
组合关系是类之间**整体与部分**的强依赖关联关系：
1. 部分类的实例完全隶属于整体类的实例
2. 二者**共存亡**
3. 当整体类的实例被销毁时，其包含的部分类实例也会随之消失，失去存在意义

---

### 【预测题5】什么是软件维护？软件维护有哪几种类型 ⭐⭐

**答题要点：**
- **定义**：软件交付使用后，为改正错误或满足新的需求而修改软件的过程

- **四种维护类型**：
  1. **改正性维护**：识别并纠正错误、优化性能
  2. **适应性维护**：适配外部环境变化
  3. **完善性维护**：满足用户新功能/性能需求
  4. **预防性维护**：提升软件可维护性、可靠性

## 六、应用题预测（2题）

### 【预测题1】程序流程图和N-S图 ⭐⭐⭐⭐⭐（必考！）

**预计考点**：画出下列伪代码对应的程序流程图和N-S图

**参考伪代码结构**：
```
输入A、B
如果 A > 0 且 B > 0
    则 执行操作1
    否则 执行操作2
循环处理直到条件满足
输出结果
```

**复习重点**：
- 顺序结构
- 选择结构（if-else）
- 循环结构（while/do-while）

---

### 【预测题2】用例图分析 ⭐⭐⭐⭐⭐（必考！）

**预计考点**：根据用例图完成下列任务

**常见问题**：
1. **图中有几个用例**（数椭圆个数）
2. **图中有几个角色**（数小人个数）
3. **给出系统的需求描述**（描述每个用例的功能）
4. **包含关系发生在哪些用例之间**（找虚线箭头连接）

**复习重点**：
- 用例（椭圆）vs 角色（小人）
- 关联关系（实线箭头）
- 包含关系（虚线箭头 + <<include>>）
- 扩展关系（虚线箭头 + <<extend>>）

## 八、考试技巧

### 单选题技巧
1. 看到"软件包括什么"→选"程序+数据+文档"
2. 看到"测试目的"→选"发现错误"
3. 看到"耦合性与独立性关系"→耦合越低独立性越高
4. 看到"瀑布模型缺点"→选"客户难以在早期看到可运行程序"

### 填空题技巧
1. UML类图三部分：**类名、属性、操作**
2. 数据流图符号：○加工、→数据流向、□数据源点/终点、=数据存储
3. 三层架构：界面层、业务逻辑层、数据访问层
4. 软件过程4活动：需求分析、设计实现、测试、演进

### 应用题技巧
1. **程序流程图**：注意箭头方向、判断框用菱形、处理框用矩形
2. **N-S图**：注意嵌套结构、每个结构一个框
3. **用例图分析**：先数椭圆（用例），再数小人（角色），再找关系
