# 使用示例

## 示例 1：固定四个文件的整理任务

### 用户输入
给你以下文件：
- `2025年4月考期真题汇总.md`
- `202510真题.md`
- `简答题.md`
- `软件工程.md`

请整理 `软件工程题目知识点汇总.md`。

### 正确处理
1. 只从：
   - `2025年4月考期真题汇总.md`
   - `202510真题.md`
   提取考点。
2. 在每个考点下补题时，可以使用四个文件里的题。
3. 来源标注：
   - `2025年4月考期真题汇总.md` -> `2504`
   - `202510真题.md` -> `2510`
   - `简答题.md` -> `笔果来源`
   - `软件工程.md` -> `课后习题`
4. `简答题.md` 和 `软件工程.md` 中如果题目本身带 `202504`、`202510`、`2504`、`2510`，必须跳过，不能再标为 `笔果来源` 或 `课后习题`。
5. 与现有汇总重复的题不再重复加入。

## 示例 2：文件列表中有多个真题文件

### 用户输入
请根据以下文件整理：
- `2024年10月真题.md`
- `2025年4月考期真题汇总.md`
- `202510真题.md`
- `简答题.md`

### 正确处理
1. 3 个文件名包含 `真题`，因此都属于考点来源文件。
2. 标注分别为：
   - `2024年10月真题.md` -> `2410`
   - `2025年4月考期真题汇总.md` -> `2504`
   - `202510真题.md` -> `2510`
3. `简答题.md` 只能补题，不能创造新考点。

## 示例 3：学科名文件的识别

### 用户输入
文件列表如下：
- `软件工程.md`
- `202510真题.md`
- `简答题.md`

### 正确处理
1. `202510真题.md`：用于提取考点，来源标 `2510`
2. `软件工程.md`：文件名是学科名，属于 `课后习题`
3. `简答题.md`：属于 `笔果来源`

### 错误处理
- 从 `软件工程.md` 单独新增一个知识点模块
- 把 `软件工程.md` 中带 `202504` 的题标成 `课后习题`

## 示例 4：重复题处理

### 场景
当前汇总文件里已经有：

```markdown
**1. 以下哪项是瀑布模型的缺点【2504】**

- A. 客户难以在早期看到可运行的程序 ✅
- B. 软件过程具有可控性和系统性
- C. 将软件过程划分为不同的阶段
- D. 需求变更容易实现
```

随后在 `软件工程.md` 中又发现同题。

### 正确处理
不再新增 `【课后习题】` 版本。

### 原因
真题来源优先级高于补充来源。

## 示例 5：同题多真题来源

### 场景
同一道题同时出现在：
- `2025年4月考期真题汇总.md`
- `202510真题.md`

### 正确处理
保留一题，合并来源：

```markdown
**1. 某题题干【2504 / 2510】**
```

不要写成两道重复题。

## 示例 6：选择题样式

### 正确格式（字母选项）

```markdown
**1. 以下哪项是瀑布模型的缺点【2504】**

- A. 客户难以在早期看到可运行的程序 ✅
- B. 软件过程具有可控性和系统性
- C. 将软件过程划分为不同的阶段
- D. 需求变更容易实现
```

### 正确格式（数字选项）

```markdown
**1. 关于测试用例的设计，下列叙述正确的是【课后习题】**

- 1. 测试用例中的输入数据必须是合法数据
- 2. 测试用例必须由程序员通过分析程序结构生成
- 3. 测试用例中应该包含程序运行的预期结果 ✅
- 4. 测试用例应该覆盖所有的输入/输出结果
```

### 错误格式

```markdown
1. 测试用例中的输入数据必须是合法数据
2. 测试用例必须由程序员通过分析程序结构生成
3. 测试用例中应该包含程序运行的预期结果 ✅
4. 测试用例应该覆盖所有的输入/输出结果
```

错误原因：缺少统一的列表样式和题目包裹格式。

## 示例 7：图片题处理

### 场景
真题文件中已经提供图片：

```markdown
![image-20260205014924637](./image-20260205014924637.png)
```

### 正确处理
在汇总文件中直接复用原图：

```markdown
**题目图片**：

![image-20260205014924637](./image-20260205014924637.png)

**答案**：直接采用题库提供的对应图示。
```

### 错误处理
- 写成“原题为作图题，自行绘制”
- 删除原图，只保留文字说明

## 示例 8：非真题文件中带考试标记的题

### 场景
`软件工程.md` 中存在：

```markdown
13. 软件工程 202504
```

### 正确处理
不能把它作为 `课后习题` 导入。

### 处理原则
- 如果该题已从真题文件导入，保留真题版本。
- 如果没有对应真题文件版本，也不要把它伪装成 `课后习题`。

## 示例 9：不删除要求

### 用户输入
不要删除原有内容，只按规则补全。

### 正确处理
1. 保留现有结构
2. 修正明显错误来源
3. 补充漏掉的真题考点
4. 在已有知识点下增补不重复的题
5. 不进行大规模删减和重构

## 示例 10：考点与题的边界

### 场景
`简答题.md` 中有一个知识点只在该文件出现，从未出现在任何 `真题` 文件中。

### 正确处理
不要新增该知识点模块。

### 允许的处理
如果它能作为某个已存在真题考点下的补充题，则可以挂到那个考点下面，并标注 `笔果来源`。
